# Content Verify Skill

通过 SerpAPI 网络检索，对新闻、科普和自媒体内容进行交叉验证并给出可信度评分。

## 配置

脚本需要 SerpAPI API Key（[serpapi.com](https://serpapi.com) 免费套餐每月 100 次）。

**方式 A — OpenClaw env 注入（推荐）：**
在 `~/.openclaw/.env` 中设置：
```
SERPAPI_API_KEY=your-key-here
```

**方式 B — skill 本地 .env：**
```bash
cp scripts/.env.example scripts/.env
# 编辑 scripts/.env，填入真实值
```

**方式 C — CLI 直接传入：**
```bash
uv run scripts/search.py --api-key your-key "搜索词"
```

可选参数（均有默认值）：
- `SERPAPI_LOCATION` — 搜索地区（默认 `Austin, Texas, United States`）
- `SERPAPI_HL` — 搜索语言（默认 `zh-cn`）
- `SERPAPI_GL` — 搜索国家（默认 `cn`）

## 脚本位置

```bash
# 安装后
uv run ~/.openclaw/skills/content-verify/scripts/search.py "<搜索词>"

# 项目内使用
uv run ./skills/content-verify/scripts/search.py "<搜索词>"
```

## 使用方法

### 网络搜索（证据收集）

```bash
uv run scripts/search.py "<搜索关键词>"
uv run scripts/search.py "<搜索关键词>" --num 5
uv run scripts/search.py "<搜索关键词>" --format json
```

### 验证流程（由 AI 主导）

**通用内容验证：**
1. 提取核心声称和关键实体
2. 执行搜索：`uv run scripts/search.py "<实体+事件关键词>"`
3. 交叉核查多个来源的一致性
4. 识别风险因素（来源缺失、情感化语言、孤证等）
5. 输出 0-100 可信度评分和等级

**新闻验证：**
1. 搜索新闻中的具体声称
2. 检查其他权威媒体是否有同样报道
3. 分析时间线一致性
4. 评估信息来源权威性
5. 给出评分：VERIFIED(90-100) / HIGH(75-89) / MEDIUM(50-74) / LOW(25-49) / FALSE(0-24)

**半真半假 / 夸大宣传型内容验证（核心流程）：**

> 很多虚假信息并非完全捏造，而是基于部分真实事实进行夸大、歪曲或断章取义。这是最难识别也最具欺骗性的类型。

1. **拆解声称** — 将内容逐条拆解为独立的事实性声称（factual claims），每条单独验证
2. **逐条搜索权威来源** — 每条声称都必须在权威数据源中独立查证：
   ```bash
   uv run scripts/search.py "<声称A的关键词> site:who.int OR site:nature.com OR site:gov.cn"
   uv run scripts/search.py "<声称B的关键词> 学术研究 OR 官方数据"
   ```
3. **对比原文与权威数据的差异** — 重点关注：
   - 数字是否被夸大（如"提升 10%" 被说成"提升 10 倍"）
   - 结论是否被过度推广（如"对特定人群有效" 被说成"对所有人有效"）
   - 前提条件是否被省略（如"在实验室条件下" 被省略）
   - 时间范围是否被篡改（如"长期趋势" 被说成"立竿见影"）
   - 因果关系是否被伪造（如"相关性" 被说成"因果关系"）
4. **生成结构化报告** — 必须使用以下格式输出：

```
## 验证报告

### ✅ 事实部分（已核实为真）
| 序号 | 原文声称 | 权威来源 | 核实结论 |
|------|----------|----------|----------|
| 1 | "XX研究发现Y成分有Z效果" | Nature 2024, DOI:xxx | 确认：该研究确实发现... |

### ⚠️ 夸大/歪曲部分
| 序号 | 原文声称 | 事实基础 | 夸大之处 | 权威来源 |
|------|----------|----------|----------|----------|
| 1 | "能治愈所有癌症" | 仅对特定细胞系有抑制作用 | 将体外实验推广为临床治愈 | PubMed PMID:xxx |

### ❌ 虚假部分（无事实基础）
| 序号 | 原文声称 | 反证据 | 来源 |
|------|----------|--------|------|
| 1 | "经FDA批准" | FDA官网无此批准记录 | fda.gov |

### 综合评分：XX/100（等级）
- 事实占比：X/Y 条声称有事实基础
- 夸大程度：[轻微/中度/严重]
- 欺骗手法：[列出使用的手法，如数据夸大、断章取义、偷换概念等]
```

**科学主张验证（自动预检伪科学）：**
1. 扫描伪科学标志：
   - 🚨 "量子" + 医学词（量子水疗、量子能量治疗）
   - 🚨 "包治百病" / "万能药" / "没有副作用"
   - 🚨 "医学界隐瞒" / "科学家不敢说"
   - 🚨 "祖传秘方" / "排毒产品"
2. 搜索权威科学机构和同行评审来源
3. 评估证据等级（一级A/B → 二级 → 三级 → 无证据 → 反证据）
4. 核查专家共识

## 权威数据源与来源筛选

### 优先采信的权威来源（白名单）

搜索结果中，**仅以下类型来源可作为核实依据**：

**科学/医学：**
- 同行评审期刊：Nature, Science, The Lancet, NEJM, JAMA, Cell, BMJ, PubMed 收录论文
- 权威机构：WHO (who.int), CDC (cdc.gov), FDA (fda.gov), EMA (ema.europa.eu), 中国国家药监局 (nmpa.gov.cn)
- 学术数据库：PubMed, Google Scholar, Cochrane Library, arXiv（预印本需标注未经同行评审）

**新闻/时事：**
- 一线通讯社：Reuters, AP, AFP, 新华社
- 主流严肃媒体：BBC, NYT, Guardian, 南方周末, 财新, The Economist
- 官方发布：政府官网 (.gov / .gov.cn)、上市公司公告、法院裁判文书

**技术/科技：**
- 官方文档：产品官网、GitHub 官方仓库、RFC 文档
- 权威分析机构：Gartner, IDC, IEEE
- 技术媒体（需交叉验证）：Ars Technica, The Verge, TechCrunch

### 不可作为核实依据的来源（须警惕）

以下来源的信息**不得单独用于支持或否定声称**：

- 🚫 **自媒体/营销号**：微信公众号（非官方认证）、百家号、头条号、知乎个人回答
- 🚫 **SEO 注水站点**：内容农场、聚合采集站、AI 生成的低质量内容站
- 🚫 **匿名/不可溯源内容**：无署名、无引用、无机构背书的文章
- 🚫 **利益关联方**：产品生产商自己的宣传材料（做核实时不可做证据，但可做声称来源）
- 🚫 **wiki 类**：百度百科、搜狗百科（任何人可编辑，不可作为最终依据；维基百科可参考其引用来源）

### 来源筛选规则

在处理搜索结果时，AI 必须遵循：

1. **先看域名再看内容** — 来自白名单域名的信息优先级高于其他来源  
2. **交叉验证至少 2 个独立权威来源** — 孤证不可定论（即使来自权威来源）  
3. **标注来源可信等级** — 每条证据需注明来源名称和可信等级（权威/可参考/存疑）  
4. **对可疑来源保持怀疑** — 如果只有非权威来源支持某个声称，应标记为"证据不足"而非确认  
5. **警惕 AI 生成的注水内容** — 如果多个不同站点出现高度相似的表述但无原始来源，可能是 AI 生成的虚假信息链

## 示例

### 验证新闻传言
```
用户：hunter alpha 是之前 openrouter 的匿名模型，后来传出是 deepseek v4，也有说是 Xiaomi MiMo-V2-Pro

AI 操作：
uv run scripts/search.py "hunter alpha openrouter anonymous model identity"
uv run scripts/search.py "hunter alpha deepseek v4 openclaw"
uv run scripts/search.py "Xiaomi MiMo-V2-Pro openrouter"
→ 综合搜索结果给出可信度评分
```

### 检验科学声称
```
用户：量子水疗能治愈慢性病

AI 操作：
⚠️ 伪科学预检：检测到"量子+医学词"标志
uv run scripts/search.py "量子水疗 科学研究 证据"
uv run scripts/search.py "quantum water therapy scientific evidence"
→ 评分 0-24（FALSE），标注反证据
```

### 验证健康建议
```
用户：维生素C大剂量服用能预防感冒

AI 操作：
uv run scripts/search.py "vitamin C high dose cold prevention clinical trial"
uv run scripts/search.py "大剂量维生素C 预防感冒 研究"
→ 根据随机对照试验证据评分
```

### 识别半真半假的夸大宣传
```
用户：某保健品宣传："哈佛大学研究证实，NMN 能逆转衰老，延长寿命 30%，已获 FDA 认证"

AI 操作：
1. 拆解为 3 条独立声称：
   - A: "哈佛大学研究证实 NMN 相关效果"
   - B: "NMN 能逆转衰老，延长寿命 30%"
   - C: "已获 FDA 认证"

2. 逐条核实（仅查权威来源）：
   uv run scripts/search.py "NMN Harvard study aging site:pubmed.ncbi.nlm.nih.gov OR site:nature.com"
   uv run scripts/search.py "NMN lifespan extension 30% clinical trial"
   uv run scripts/search.py "NMN FDA approval site:fda.gov"

3. 输出结构化报告：
   ✅ 事实部分：哈佛 David Sinclair 团队确实发表过 NMN 动物实验论文（Cell, 2018）
   ⚠️ 夸大部分："延长寿命 30%" 来自小鼠实验，未在人体试验中证实；"逆转衰老"过度简化了研究结论
   ❌ 虚假部分：FDA 从未批准 NMN 作为药物，NMN 在美国作为膳食补充剂销售
   → 评分 30/100（LOW），手法：真实研究 + 数据外推 + 虚构背书
```

### 案例：AI作用被夸大与关键信息隐藏
```
用户：AI的意义具象化了 #AI #狗狗 #杂谈
万万没想到，让AI给狗看病，结果它直接“造”出了狗的抗癌药。
保罗的狗罗西得了肥大细胞癌，经历手术和化疗仍无效。在兽医宣判仅剩1-6个月生命后，保罗打开ChatGPT寻求帮助。AI给出关键建议：先做基因测序，将罗西的正常DNA与肿瘤DNA对比，找到突变点。
于是保罗自费3000澳元完成了全基因组测序，拿到了320GB的原始数据（相当于70万页、20层楼高）。他将这些数据喂给AlphaFold等AI工具，半小时内AI就找到了突变蛋白，并预测出能激活免疫系统的“新抗原”。
保罗拿着方案去找药厂，却被拒绝为单只狗生产药物。ChatGPT再次提供思路：改用mRNA疫苗技术，只给免疫系统“图纸”，让它自己识别并攻击癌细胞。
他联系新南威尔士大学RNA研究所，在不到两个月内成功制造出全球首支为狗量身定制的个性化mRNA癌症疫苗。经过伦理审批和合规流程后，2025年12月罗西接受了首针注射。
奇迹发生了：接种后肿瘤迅速缩小，三个月内其中一个肿瘤缩小近一半，罗西重新恢复活力，甚至能在公园里跳跃追兔子。

AI 操作：
1. 拆解声称：
   - A: "AI 直接造出了狗的抗癌药"
   - B: "AI 给出关键建议，半小时内找出突变蛋白和新抗原"
   - C: "保罗只花 3000 澳元就完成了全部流程"
   - D: "疫苗是 AI 研制的"
   - E: "疫苗研发周期极短，效果显著"
2. 逐条查证：
   - 查找权威报道/论文，明确疫苗研发的实际主导者、AI 在流程中的真实作用、实验室和科学家团队的贡献、成本与资源门槛
   - 检查是否有信息被有意隐去（如顶尖实验室、科学家资源、伦理审批、资金投入等）
3. 结构化报告：
   ✅ 事实部分：AI 工具（如 AlphaFold）可辅助蛋白结构预测，基因测序和新抗原筛选在科研中已应用；mRNA 疫苗技术成熟，部分实验室具备快速定制能力。
   ⚠️ 夸大/歪曲部分：AI 仅为辅助分析工具，核心研发和疫苗制备由专业团队完成；疫苗研发依赖顶级实验室和科学家资源，普通人难以复制；3000 澳元仅为测序费用，实际研发成本远高于此。
   ❌ 虚假部分：疫苗“由AI直接研制”不符实际，AI 并不能独立完成药物开发和生产。
   → 评分 40/100（LOW），手法：AI 作用夸大、关键信息隐藏、资源门槛淡化。

【批判性验证要点】
- 对“AI 造药”“AI 研制疫苗”等表述，必须追溯实际研发流程，明确 AI 仅为工具，主导为科学家团队
- 检查是否有意隐去高门槛资源（顶级实验室、专家、伦理审批、资金等），警惕“平民神话”叙事
- 关注成本、周期、资源等被淡化或省略的细节，避免被营销话术带偏
- 结论：AI 在生物医药领域有重要辅助价值，但不能神化其作用，更不能忽略科学家和实验室的主导地位
```

## 评分标准

| 评分 | 等级 | 说明 |
|------|------|------|
| 90-100 | ✅ VERIFIED | 已被多个权威来源独立验证 |
| 75-89  | 🟢 HIGH | 来源可信，事实基本准确 |
| 50-74  | 🟡 MEDIUM | 有限证据或部分有争议 |
| 25-49  | 🟠 LOW | 缺乏权威来源支持 |
| 0-24   | 🔴 FALSE | 明显虚假或已知伪科学 |

## 重要提示

- **未经网络检索不得直接给出高置信度结论**
- 若检索证据不足，明确标注"证据不足/待进一步核实"
- 搜索结果仅作为证据候选，不等于事实结论
- 对争议内容建议多关键词、多角度搜索
- **对部分真实、部分夸大的内容，必须逐条拆解，分别标注事实部分和夸大部分**
- **核实依据仅采信权威来源，忽略自媒体、营销号和 SEO 注水站点**
- **当内容包含真实信息时，不可因部分属实就给高分——夸大部分必须单独指出并扣分**
